About Git Aliasing

If you want cool aliases like

$ git co { branch_name }

to replace:

$ git checkout { branch_name }

You need to edit your .gitconfig file in your root directory!

To do so, run this command: 

$ vim ~/.gitconfig

The "~" symbol stands for the root directory, so you could run this command from whatever folder you are in!

my .gitconfig looks like this: 

[user]
        name = Eric
        email = ericso0303@gmail.com
[core]
        autocrlf = false
        safecrlf = true
        editor = vim
[alias]

diffIndex = diff --cached  
gap = add --patch  
superStash = stash --include-untracked --keep-index  
hist = log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit --date=relative

[filter "lfs"]
        clean = git-lfs clean %f
        smudge = git-lfs smudge %f
        required = true

Under the [alias] section you can see I have diffIndex, gap, superStash and hist set up for cool commands that I usually forget about. You can just add in any command you'd like in this list.

if you want to add "git checkout" as "git co" you would change the file to:

//I cut stuff out of the top
[user]
        name = Eric
        email = ericso0303@gmail.com
[core]
        autocrlf = false
        safecrlf = true
        editor = vim
[alias]

co = checkout  
diffIndex = diff --cached  
gap = add --patch  
superStash = stash --include-untracked --keep-index  
hist = log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit --date=relative

[filter "lfs"]
        clean = git-lfs clean %f
        smudge = git-lfs smudge %f
        required = true

and just save it! (for vim noobs like me type ":wq!")
