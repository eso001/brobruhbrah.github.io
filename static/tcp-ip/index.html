TCP/IP

...

protocol means a set of rules two sides agree on to use when communicating. Like two people from different countries choosing to speak english.

Client server model:
client - asks for services
server - performs services

client sends requests to server and receives a response in return.

So how is the information actually being transferred??

When the browser and web server are connected. Each one receives an IP address.

IP stands for internet protocol

The sequence of numbers identifies computers together.
When you make a connection, your server makes a socket, which is a line across which the information flows.

all protocols are about how information needs to be structured
HTTP(web) FTP(file) SMTP(mail)
they all go from computer to computer using IP. We need a protocol for how to send information not just how it's structured.

TCP is HOW to send information. The act of separating the information into packets is called TCP

IP is Where to send.

HTTP/FTP is what you send, or how to strucutre it.

You can use node to send email server, file server or anything.

TCP looks a lot like a stream. Node treats the information coming down this pipeline as a stream.

Sockets, real time communication. We open and close sockets constantly online.
