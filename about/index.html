What you need to know about Index in Git:

Index is one of the git repositories that manages your code. Also called the "staging areas"

There are three main repositories that relate to this.

Workplace --> Index --> local repo --> remote repo

When you type code directly into your editor you are typing in your "Workplace" repo.

Then you can go to your command line and update your Index by performing a 

//this just updates the Index with tracked files
git add -u

You can use "git status" to see what files are pushed to the index 

ericso0303@MacBook-Pro-de-eric:~/Desktop/Raffle(feat/raffleLayout-part2⚡) » git status  
On branch feat/raffleLayout-part2  
Changes to be committed:  
  (use "git reset HEAD <file>..." to unstage)

    modified:   client/src/index.js
    modified:   client/styles/css/partials/base.css
    modified:   client/styles/css/style.css
    modified:   client/styles/sass/partials/base.scss
    modified:   client/webpack.config.js

Changes not staged for commit:  
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

    modified:   client/webpack.config.js

Untracked files:  
  (use "git add <file>..." to include in what will be committed)

    .vscode/
    client/npm-debug.log
    client/src/assets/
    client/src/helpers/
    client/src/test.js
    client/styles/css/partials/landing.css
    client/styles/sass/partials/landing.scss
    output.js
    output.json
    work.js

So everything that is staged for the commit is in the "Index" and everything else that is unstaged is in your "Workplace".

So your "Index" holds all the staging information before you commit your code. Some people do not even use Index in their git workflow but it can really be useful if you want to divide your code up into multiple commits so others can more modularly read through your code!

all you have to do now is git commit to move the code to your local repository

git commit -m "I love writing commit messages"

Then a git push to move it up to the remote repo

git push origin master

And thats how you get your code all the way from your workplace to the remote repo using the Index!

What should you have learned from this post:
The Index is a repo to store the state of your code. More importantly, it can act as a medium between your Workplace and your local repo so you can easily manage what goes to your local repo and keep your commit history beautiful.
